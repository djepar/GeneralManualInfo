General Networking

Introduction to Network

Network categorized based on their geographic span

Personal Area Network (Pan)
A PAN is the smallest type of network and has a limited span, interconnecting personal devices such as those that are Bluetooth enabled.
Local Area Network (LAN)
A LAN is a network commonly used to interconnect and share computer resources inside a building or multiple buildings in a limited area.
Campus Area network (CAN)
A CAN -- often called simple an enterprise network -- spans multiple buildings in a campus environment such as a university or another large organization.
Metropolitan area network (MAN)
A MAN spans multiple buildings in a city area
Wide area network (WAN)
A WAN is much larger than the other network types and can span many areas such as cities, states or countries.

Network Topologies

Topology : the architecture of a network

Token-Ring Topology

Works through the passing of a token around the ring.

It's a deterministics network, meaning each station connected to the network is ensured access for transmission of its message at regular or fixed intervals.

Disadvantages : IP error message, token may stall, must break circle to add/remove a device.

Star Topology

Most common networking Topology in today's LANs.

All the devices are connected to a hub or a switch. They act as a multiport reader : "which means the data it receives is broadcast and seem by all devices connected to its ports."

Bus Topology

The computers share the media for data transmission through a coaxial cable.

Mesh Topology

All networking devices are all connected to each other. It's a costly installation, but useful for high-reliability application.

The OSI Model

Overview of the OSI Model

The OSI Model Contains seven layers that describe networking functions from the physical network interface to the software application interfaces.

At each layer, they are different protocol that operate.

Each layer performs data encapsulation by putting its own message format or header onto the data as it is being passed down from layer 7 to layer 1 for transmission.
Layer Function Examples
7. Application Provides support for applications. HTTP, FTP, SMP(email)
6. Presentation Handle protocol conversation and data translation
(translation, compression, encryption, decryption)		 ASCII, JPEG, SSL
5. Session Established, manages and terminates sessions NFS, SQL, API
4. Transport Handles end-to-end delivery to ensure error-free packets. TCP, UDPP
3. Network Provides addressing and routing decisions IP, ICMP
2. Data Link For the flow of data Examples
1. Physical Handles signals and media NICs, twisted-pair cable, fiber, wireless

1. Physical Layer

Electrical and mechanical connection to the network

Equipments:

  • Unshielded twisted-pair(UTP)
  • Fiber, network interface card(NICs)

2. Data-Link Layer

Handle errory recovery, flow control (synchronization) and sequencing (for receiving and sending). Sometime call the Media Access Control Layer (MAC layer) because it's where the MAC addressing is defined.

Protocol
Ethernet 802.3 standard
Addressing
MAC address

Network Layer

Accepts outgoing messages which are combines or segmented into packts. A header is added with routing information

Protocol : Internet Protocol

Transport Layer

The end-toend delivery between devices. Primary goald is the integrity of the message between the source and the destination. Reassemble the segments and packets and handle the flow control.

Protocol
Transmission Control Protocol (TCP) or User Datagram Protocol (UDP) and more...

Session Layer

This layer provides the control functions necessary to established, manage and terminate connections, as required, to satisfy user requests. (Beasley, Nilkaew 2014. p. 13)

Technologies
Network File System (NFS), Structured Query Language (SQL)

Presentation Layer

This layer accepts and structures messages for an application.(Beasley, Nilkaew 2014. p. 13-14)Can also translates the message from one code to another. Encrypt and compress data.

Technologies
American Standard Code for Information Interchange(ASCII) and Joint Pohotographic Expert Group (JPEG)

Application Layer

this layer intereacts with application programs that incorporate a communication component such as an Internet browser and email. This layer is responsible for logging in the message, interpreting the request, and determining what information is needed to support the request.

  • HyperText Transfer Protocol (HTTP) --> web browser
  • File Transfert Protocol (FTP) --> transfering files
  • Simple Mail Transfer Protocol (SMTP) --> email transmission
